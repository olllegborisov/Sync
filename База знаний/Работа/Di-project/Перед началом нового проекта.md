модификаторы можно называть коротко и с двумя дефисами 
![[Pasted image 20250721171701.png]]


Стараться использовать fit-content вместо max-content, так как max-content будет увеличивать максимально ширину и при переносе секции, может что то поехать например. fit-content занимает максимально доступную часть пространства и не вылезет за пределы родительского блока

для секций с цифрами использовать счетчики https://doka.guide/css/css-counters/
![[Pasted image 20250723181111.png]]

Для повторяющихся элементов через раз, например один элемент черный другой белый можно использовать nth-child(even) (выбирает все четные числа)

для того что бы сделать каждый допустим третий элемент, можно использовать nth-child(2n + 1) 

Для того что бы использовать стиль к определённому классу, и  после указанного, нужно использовать тильду (~)
block1 ~ block2 — назначить стиль ко всем блокам block2 после block1
![[Pasted image 20250723181704.png]]
то есть можно добавить плюсы всем последующим блокам после первого блока.

Так же можно использовать тильду для того что бы задавать отступы. И не прописывать последнему блоку last-child {margin-bottom: unset;}


Использовать спрайты только когда это необходимо, использовать псевдоэлементы с svg когда требуется масштабируемость верстки. Например приме выше с плюсом. 

Использовать короткие названия классов, и модификаторов. А так же использовать модификаторы при крайней необходимости.

Принцип чем меньше классов тем лучше. Тем проще будет бэку на натяжке.
